# Contributing to Consul MCP Server

First off, thank you for considering contributing to this project! It's people like you that make the open source community such a great place to learn, inspire, and create.

## How to Contribute

### 1. Reporting Bugs
If you find a bug, please create an issue in the GitHub repository. Include:
- A clear and descriptive title
- Steps to reproduce the issue
- Expected vs. actual behavior
- Your environment (OS, Node.js version, Consul version)

### 2. Suggesting Enhancements
We welcome new features! If you have an idea:
- Open an issue explaining the feature and why it would be useful
- If you can, provide examples or mockups

### 3. Pull Requests
If you want to contribute code:
1. Fork the repository
2. Create a new branch (`git checkout -b feature/amazing-feature`)
3. Make your changes and test them
4. Commit your changes (`git commit -m 'feat: Add amazing feature'`)
5. Push to the branch (`git push origin feature/amazing-feature`)
6. Open a Pull Request

## Development Setup

1. Clone the repo
2. Install dependencies: `npm install`
3. Make your changes in the `src/` directory
4. Build the project: `npm run build`
5. Test locally using the MCP inspector or your preferred IDE

Please make sure your code follows the existing style and passes the TypeScript compilation (`npm run build`).
